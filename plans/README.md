## Findings considered but NOT planned (selected out by the maintainer)

Kept here so they're not re-discovered as new. Each was a real, vetted finding;
the maintainer chose to defer it this round.

- **Debug-log noise + hygiene**: ~74 `console.log/info` across `src/`
  (heaviest in `table-size-buttons.ts` with emoji logs, plus `drag-to-resize.ts`
  preview logs). Also: `clean` script uses Unix `rm -rf` on a Windows project
  (`package.json:17`); a stray `"//will run and exit": ""` key
  (`package.json:18`); dead `src/cell-contents-export.ts` (re-export with no
  importers); unbounded `data-span-x/y` (clamped to ≥1 but not to grid size,
  `table-renderer.ts:131-133`). Low-risk S cleanup, deferred.
- **CI + one-command e2e**: no `.github/workflows/` at all; no typecheck/test
  gate on push; add a Playwright `webServer` block so e2e runs unattended.
  Deferred.
- **Render resilience to malformed `data-*`**: `parseJSONAttr`
  (`table-model.ts:22-29`) rethrows on bad JSON, so one malformed
  `data-edges-h/v` attribute throws out of the whole render. Degrade gracefully
  (warn + fallback) instead. Deferred (needs a fallback-behavior decision).
- **Dependency security bumps**: `pnpm audit` flags `happy-dom@15` (bump to
  ≥20.8.9), `vite@6.3.5` (→ ≥6.4.2), `@playwright/test@1.55.0` (→ ≥1.55.1) —
  all dev-only, no production exposure. Deferred.
- **Harden demo dev API path handling**: `demo/api.ts` / `demo/example-api.ts`
  file-write endpoints validate `..` with string checks that are incomplete on
  Windows mixed separators; replace with `path.resolve` + containment check.
  Dev-only exposure. Deferred.
